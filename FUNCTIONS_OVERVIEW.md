# Описание функций и классов проекта Test_RK

---

## app.py
**Назначение:** Основное Flask-приложение для отображения результатов проверок.

### Классы
- **Inspections(db.Model)**
  - Модель ORM для таблицы inspections в PostgreSQL.
  - Поля: id, entity_name, ogrn, purpose, status, result, inspection_date.
  - Используется для работы с результатами проверок через SQLAlchemy.

### Функции
- **index()**
  - Маршрут `/` (главная страница).
  - Получает страницу и размер страницы для пагинации.
  - Загружает данные из таблицы inspections, формирует объект пагинации.
  - Рендерит шаблон `index.html` с данными и пагинацией.

---

## parser_v3.py
**Назначение:** Парсер для получения данных о проверках с API Госуслуг.

### Функции
- **fetch_inspections(page=1, cookie_manager=None)**
  - Получает данные о проверках за последний месяц с помощью POST-запроса к API.
  - Использует cookies для авторизации (через cookie_manager).
  - Возвращает JSON-ответ с результатами или None при ошибке.

- **format_status(item)**
  - Форматирует статус проверки для отображения.
  - Используется для приведения статусов к человекочитаемому виду.

- **format_result(item)**
  - Форматирует результат проверки (выявлены/не выявлены нарушения).
  - Используется для отображения результата в карточке.

- **save_to_json(data, filename='inspections.json')**
  - Сохраняет полученные данные в файл inspections.json.
  - Используется для передачи данных в init_db.py.

- **main()**
  - Основная точка входа: инициализирует CookieManager, собирает все страницы данных, сохраняет их в JSON.
  - Запускается при старте скрипта.

---

## init_db.py
**Назначение:** Инициализация базы данных и загрузка данных из inspections.json.

### Функции
- **init_database()**
  - Подключается к базе PostgreSQL по DATABASE_URL.
  - Создаёт таблицу inspections, если не существует.
  - Очищает таблицу и сбрасывает sequence.
  - Загружает данные из inspections.json (или /tmp/inspections.json).
  - Вставляет все записи в таблицу.
  - Очищает файл после загрузки.
  - Используется для полной перезагрузки данных в БД.

---

## cookie_manager.py
**Назначение:** Получение и управление cookies для доступа к API Госуслуг.

### Классы
- **CookieManager**
  - **__init__()** — инициализация, настройка путей к файлам cookies.
  - **setup_chrome_options()** — настройка параметров Chrome для Selenium (работа в Docker).
  - **get_cookies_with_selenium()** — получение cookies через Selenium, эмулируя браузер.
  - **save_cookies(cookies_dict, expiry_hours=24)** — сохранение cookies в файл с временем жизни.
  - **load_cookies()** — загрузка cookies из файла, проверка срока действия.
  - **get_fresh_cookies(force_refresh=False)** — получение свежих cookies (сначала из файла, при необходимости — через Selenium).
  - **generate_headers(cookie_string)** — генерация HTTP-заголовков для запросов с cookies.

---

## scheduler.py
**Назначение:** Планировщик для автоматического обновления данных по расписанию.

### Функции
- **run_parser_and_update_db()**
  - Запускает parser_v3.py для получения новых данных.
  - После успешного парсинга запускает init_db.py для загрузки данных в БД.
  - Логирует все этапы и ошибки.

- **main()**
  - Настраивает логирование.
  - Добавляет задачу на ежедневный запуск парсера и загрузки данных в 6:00 утра.
  - Запускает бесконечный цикл ожидания расписания.

---

## start_with_scheduler.sh
**Назначение:** Скрипт запуска контейнера и всех сервисов в правильной последовательности.

### Логика
1. Ожидание запуска PostgreSQL (sleep 15)
2. Запуск парсера (parser_v3.py)
3. Загрузка данных в БД (init_db.py)
4. Запуск Flask-приложения (app.py) в фоне
5. Запуск планировщика (scheduler.py) в основном процессе

---

## templates/index.html
**Назначение:** HTML-шаблон для отображения результатов проверок.
- Использует Bootstrap для оформления.
- Выводит карточки с результатами проверок и пагинацию.
- Использует переменные Jinja2: checks (список проверок), pagination (объект пагинации).

## requirements.txt
**Назначение:** Список всех зависимостей Python для работы проекта.
