# Test_RK - Система отображения результатов проверок

Flask-приложение для отображения результатов проверок с использованием PostgreSQL.

## Требования

- Docker
- Docker Compose

## Быстрый запуск

### Обычный режим (без планировщика)
1. **Клонируйте репозиторий и перейдите в директорию проекта:**
   ```bash
   cd Test_RK
   ```

2. **Запустите приложение с помощью Docker Compose:**
   ```bash
   docker-compose up --build
   ```

3. **Дождитесь завершения инициализации:**
   - Парсер получит данные с API и сохранит в `inspections.json`
   - Данные будут загружены в PostgreSQL
   - Flask-приложение запустится

4. **Откройте браузер и перейдите по адресу:**
   ```
   http://localhost:5000
   ```

### Режим с планировщиком (автоматическое обновление)
1. **Запустите приложение с планировщиком:**
   ```bash
   docker-compose -f docker-compose.scheduler.yml up --build
   ```

2. **Планировщик будет:**
   - Запускать парсер каждый день в 6:00 утра
   - Автоматически обновлять базу данных
   - Логировать все операции в `/app/scheduler.log`

**Примечание:** Первый запуск может занять несколько минут, так как парсер получает данные с внешнего API.

## Структура проекта

- `app.py` - основное Flask-приложение
- `parser_v3.py` - парсер для получения данных с API (автоматически запрашивает данные за последний месяц)
- `scheduler.py` - планировщик для периодического запуска парсера
- `init_db.py` - скрипт инициализации базы данных
- `inspections.json` - данные для загрузки в базу (создается парсером)
- `templates/index.html` - HTML-шаблон для отображения результатов
- `requirements.txt` - зависимости Python
- `Dockerfile` - конфигурация Docker-образа
- `docker-compose.yml` - конфигурация Docker Compose

## Конфигурация

### Переменные окружения

Приложение использует следующие переменные окружения:

- `DATABASE_URL` - URL подключения к PostgreSQL (по умолчанию: `postgresql://postgres:postgres@postgres:5432/Test_RK`)
- `FLASK_ENV` - окружение Flask (по умолчанию: `production`)

### База данных

- **База данных:** Test_RK
- **Пользователь:** postgres
- **Пароль:** postgres
- **Порт:** 5432

## Управление контейнерами

### Запуск в фоновом режиме
```bash
docker-compose up -d
```

### Остановка
```bash
docker-compose down
```

### Просмотр логов
```bash
docker-compose logs -f app
```

### Пересборка образа
```bash
docker-compose up --build
```

### Обновление данных
```bash
# Обновление данных через парсер
docker-compose exec app python update_data.py
```

### Настройка расписания парсера
Для изменения частоты запуска парсера отредактируйте файл `scheduler.py`:

```python
# Запуск каждый день в 6:00 утра (по умолчанию)
schedule.every().day.at("06:00").do(run_parser)

# Запуск каждый час
schedule.every().hour.do(run_parser)

# Запуск каждые 6 часов
schedule.every(6).hours.do(run_parser)

# Запуск каждые 12 часов
schedule.every(12).hours.do(run_parser)
```

### Очистка данных
```bash
docker-compose down -v
```

## Разработка

### Локальная разработка без Docker

1. Установите PostgreSQL
2. Создайте базу данных `Test_RK`
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустите инициализацию базы данных:
   ```bash
   python init_db.py
   ```
5. Запустите приложение:
   ```bash
   python app.py
   ```

## Возможные проблемы

### Проблема с подключением к базе данных
Убедитесь, что PostgreSQL контейнер полностью запустился перед запуском приложения. Приложение автоматически ждет 15 секунд перед попыткой подключения.

### Проблема с парсером
Если парсер не может получить данные, проверьте:
- Доступность интернета в контейнере
- Корректность API-запросов
- Наличие необходимых заголовков в запросах

### Проблема с правами доступа к файлам
Если возникает ошибка "Read-only file system", приложение автоматически сохраняет файлы в временную директорию `/tmp` и затем копирует их в основную директорию.

### Проблема с правами доступа
Если возникают проблемы с правами доступа к файлам, убедитесь, что Docker имеет необходимые разрешения.

### Проблема с портами
Если порт 5000 занят, измените маппинг портов в `docker-compose.yml`:
```yaml
ports:
  - "8080:5000"  # Используйте порт 8080 вместо 5000
```
