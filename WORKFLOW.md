# Логика работы системы Test_RK

## Последовательность запуска

### 1. Получение заголовков (`parser.py`)
- Использует Selenium для автоматизации браузера
- Переходит на сайт Госуслуг и выполняет поиск
- Перехватывает реальные HTTP-заголовки запросов
- Сохраняет заголовки в файл `real_headers.json`

### 2. Парсинг данных (`parser_v3.py`)
- Загружает заголовки из `real_headers.json`
- Обновляет Request-GUID для каждого запроса
- Выполняет запросы к API Госуслуг с реальными заголовками
- Обрабатывает вложенную структуру данных
- Сохраняет результаты в `inspections.json`

### 3. Загрузка в БД (`init_db.py`)
- Подключается к PostgreSQL
- Создает таблицу `inspections` если не существует
- Загружает данные из `inspections.json` в базу данных
- Обрабатывает дубликаты и ошибки

### 4. Запуск веб-сервера (`app.py`)
- Запускает Flask-приложение
- Отображает данные из базы данных на веб-странице
- Предоставляет интерфейс для просмотра результатов проверок

### 5. Планировщик (`scheduler.py`)
- Запускается в фоне
- Ежедневно в 6:00 повторяет шаги 1-3
- Логирует все операции

## Обработка ошибок

Система включает проверки на каждом этапе:
- Проверка успешности получения заголовков
- Проверка наличия файла `real_headers.json`
- Проверка успешности парсинга данных
- Проверка загрузки данных в БД
- Проверка наличия записей в БД

При любой ошибке процесс останавливается с информативным сообщением.

## Файлы данных

- `real_headers.json` - реальные заголовки HTTP-запросов
- `inspections.json` - данные о проверках в формате JSON
- `scheduler.log` - логи работы планировщика

## Автоматизация

В Docker-контейнере все процессы запускаются автоматически через `start_with_scheduler.sh`:
1. Ожидание запуска PostgreSQL (15 секунд)
2. Получение заголовков
3. Парсинг данных
4. Загрузка в БД
5. Проверка данных
6. Запуск веб-сервера
7. Запуск планировщика
